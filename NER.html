NER - Name Entity Recognition

Named Entity Recognition (NER) is a natural language processing (NLP) technique that involves identifying and extracting entities of interest from unstructured text. These entities can include people's names, locations, organizations, dates, times, and more.

The process of NER involves breaking down a piece of text into smaller units called tokens and then analyzing these tokens to determine whether they represent an entity. This is typically done using machine learning algorithms that are trained on large amounts of annotated text data.

NER can be appllied in many real-world situations when analyzing a large quantity of text is helpful. Some examples of NER includes:
- Improve customer support by categorizing and filtering user requests, complaints, and questions. It can help businesses obtain insights about their customers.
- Help categorize applicants’ CVs and speed up the process.
- Improve search and recommendation engines using recognized entities.
- Search and extract useful information from documents and blog posts.

In order to comprehend the Named Entity Recognition (NER) process in Natural Language Processing (NLP), it would be beneficial to begin by grasping the definition of a Named Entity.

Named Entity

Named entities are proper nouns that refer to specific entities that can be a person, organization, location, date, etc. Consider this example – “Mount Everest is the tallest mountain”. Here Mount Everest is a named entity of type location as it refers to a specific entity.

Named Entity Recognition

Named Entity Recognition (NER) is a technique used in information retrieval and natural language processing to extract named entities from text.
This process involves two steps:
The first step is to perform Part of Speech (POS) tagging on the text.
The second step involves using the information from the POS tagging to identify and extract the named entities from the text.

Ambiguity in Named Entity

Although the category of a term may be clear for a person, there can be ambiguity in classification when it comes to computers. Let's examine a few examples of such ambiguity:

1.  The phrase "England won the 2019 world cup" could refer to the Organisation named England or the Location of England where the event took place.

2.  Similarly, the phrase "Washington is the capital of the US" could refer to the Location named Washington or the Person who was the first president of the US and also named Washington.

Methods of NER

1.  A potential approach is to utilize various machine learning algorithms to train the model for multi-classification. However, this method necessitates extensive labelling efforts. Furthermore, due to the ambiguity present in sentences, the model must possess a profound contextual comprehension to tackle this issue. This, in turn, makes it an arduous task for a basic machine learning algorithm.

2.  An alternative method is to utilize Conditional Random Field (CRF) implemented by NLP Speech Tagger and NLTK. This is a probabilistic model that is effective in modeling sequential data such as words. CRF is able to capture a profound understanding of the sentence's context. In this model, the input

X = {x1→, x2→, x3→, ......, xT→}

3.  The NER approach based on deep learning is more precise than previous methods as it can group together words effectively. This is because it employs a technique called word embedding, which can comprehend the meaning and structure of words in relation to one another. Additionally, it has the ability to learn and examine both subject-specific and high-level words automatically. As a result, deep learning NER is useful for performing multiple tasks. It can also perform most of the repetitive tasks autonomously, freeing up researchers' time to use more productively.
